4. Data Storage and Security

4.1 Firebase and Google Cloud

All your business data is stored in Google Firebase Firestore, a cloud database operated by Google LLC. Data is stored in Google's servers (Mumbai / Asia-South1 region by default).

  • Access to your data is restricted by Firestore Security Rules — only you (authenticated with your UID) can read or write your data.
  • Data is encrypted at rest and in transit (TLS 1.2+).
  • Firebase Authentication manages your login credentials — your password is never stored in plain text.

Google's privacy policy: policies.google.com/privacy

4.2 Local Storage

Some settings and cached data are stored locally on your device using the browser's localStorage. This data never leaves your device except via Firestore sync.

4.3 Security Measures

  • HTTPS-only access (Firebase Hosting enforces this)
  • Content Security Policy (CSP) headers restrict script execution
  • Auth rate limiting to prevent brute-force attacks
  • Input sanitization throughout the app
  • Firestore rules block all access except from the authenticated user

6. Cookies and Local Storage

Stratix is a PWA and does not use third-party tracking cookies. The App uses:

  • localStorage: Stores your session token, language preference, dark mode setting, and cached data for offline use.
  • Service Worker Cache: Stores app shell files (HTML, icons, fonts) for offline access.
  • Firebase SDK cookies: Google Firebase may set a session cookie for authentication. This is strictly necessary for the login function.

No advertising cookies, no cross-site tracking, no analytics pixels.
